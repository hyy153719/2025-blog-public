1.模型精度与大小 (通常决定你需要多好的显卡)

fp16 (Half Precision) / fp32 (Full Precision)：

意思： 代表“浮点数精度”。fp32 是全精度（算得最准，文件最大，最吃显存）；fp16 是半精度。
正常情况fp16够用

bf16 (Bfloat16)：

 另一种专门为深度学习优化的半精度格式。如果你看到它，通常意味着模型针对较新的显卡（如 RTX 30/40 系）有更好的加速效果。

pruned (剪枝版)：

作者在训练完大模型后，删除了模型中那些“对最终画图效果没啥用”的冗余数据。
它的体积通常只有原版的一半，但画图效果几乎一模一样。

emaonly (仅指数移动平均)：

 EMA 是一种在训练时让模型更稳定的算法。带有 emaonly 意味着这个模型只保留了画图需要的数据，去掉了那些“如果别人想继续用它训练新模型”才需要的数据。
只用来生图可以选，不能用来炼丹

2.文件格式 (决定了安全性与加载方式)

.safetensors：

大多数绘图模型的文件格式， 加载速度极快，且安全

.ckpt / .bin / .pt：

 较老的深度学习模型打包格式。

尤其是 .ckpt（Checkpoint），因为它可以包含任意可执行代码，现在正逐渐被淘汰。如果同一个模型有 .safetensors 和 .ckpt 两个版本，永远优先选前者。

3.视觉与架构词汇 (决定了模型处理信息的通道)
ViT (Vision Transformer)：

视觉 Transformer。代表模型是用切图块的方式来理解图像的

-l / -h / -g (尺寸代号)：

通常跟在 ViT 后面，如 vit-h。代表视觉提取模块的规模：-l (Large 大型)、-h (Huge 超大型，目前的高清主流)、-g (Giant 巨型)。

CLIP：

这是一个把“文字”和“图片”联系起来的关键模型。AI 之所以能听懂你的提示词，就是靠 CLIP 把人类语言翻译成视觉特征。

4.文本语言模型专有参数 (LLM)

7B / 13B / 70B 等：

B 代表 Billion（十亿）。7B 就是指这个模型有 70 亿个参数。参数越大，模型越聪明，但对电脑硬件的要求也呈指数级上升。

Chat / Instruct：

带有这种后缀的模型，说明它经过了特殊的“聊天微调”或“指令微调”。它能像正常人一样一问一答。如果不带，它可能只是一个只会疯狂续写句子的基础大模型 (Base Model)。

GGUF / AWQ / GPTQ：

这些都是大语言模型的“量化（压缩）格式”。它们能把一个原本需要极其昂贵专业显卡才能跑的千亿级大模型，压缩到家用电脑甚至手机上也能运行。
